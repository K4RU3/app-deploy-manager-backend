# Self-Hosted App Deployment Manager

## Backend Implementation Specification (Node.js)

---

# 1. Project Overview

This backend is the **control system for a self-hosted application platform**.
It automatically deploys, updates, and manages applications using:

* Git repositories
* Docker containers
* nginx reverse proxy
* TLS certificates
* Docker volume backups

Applications are defined **inside their own repository** using a configuration file.
The manager reads this configuration and deploys the application automatically.

The backend acts as an **orchestrator for infrastructure**.

# 6. Application Repository Specification

Every managed application repository **must contain a configuration file**.

Required file:

```
app-service.json
```

Example:

```
{
  "name": "blog",
  "domain": "blog.example.com",
  "containerPort": 3000,
  "volume": "blog_data",
  "autoBackup": true
}
```

Required fields

| Field         | Description          |
| ------------- | -------------------- |
| name          | service name         |
| domain        | public domain        |
| containerPort | internal docker port |
| volume        | docker volume        |
| autoBackup    | enable deploy backup |

Required files in repository

```
Dockerfile
app-service.json
```

```

```

4 register service

# 10. nginx Service

Responsibilities

* generate reverse proxy configuration
* enable / disable service
* reload nginx safely

Configuration structure

```
/etc/nginx

system/
managed/

sites-enabled/
```

The manager **must only modify the managed directory**.

---

## nginx template

```
server {
    listen 443 ssl;
    server_name {{domain}};

    ssl_certificate /etc/letsencrypt/live/{{domain}}/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/{{domain}}/privkey.pem;

    location / {
        proxy_pass http://{{containerName}}:{{port}};
    }
}
```

# 12. Backup Service

Backup method

```

   ↓

   ↓

   ↓

Backup format

```

```

Backup location

```
